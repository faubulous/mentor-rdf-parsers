# N3 (Notation3) Parsing

Parse [W3C N3](https://w3c.github.io/N3/spec/) documents with full CST access and RDF/JS quad generation.

## Basic Usage

```typescript
import { N3Lexer, N3Parser, N3Reader } from '@faubulous/mentor-rdf-parsers';

const input = `
  @prefix ex: <http://example.org/> .
  ex:Alice ex:knows ex:Bob .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const quads = new N3Reader().visit(cst);
```

## Formulas (Graph Literals)

N3 formulas are graph literals enclosed in `{ }`:

```typescript
import { N3Lexer, N3Parser, N3Reader } from '@faubulous/mentor-rdf-parsers';

const input = `
  @prefix ex: <http://example.org/> .
  { ex:Alice ex:knows ex:Bob } => { ex:Bob ex:knows ex:Alice } .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const quads = new N3Reader().visit(cst);
```

## Implications (Rules)

N3 supports logical implications using `=>`:

```typescript
const input = `
  @prefix ex: <http://example.org/> .
  @prefix log: <http://www.w3.org/2000/10/swap/log#> .
  
  { ?x ex:parent ?y . ?y ex:parent ?z } => { ?x ex:grandparent ?z } .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const quads = new N3Reader().visit(cst);
```

## Quick Variables

N3 quick variables use the `?` prefix:

```typescript
const input = `
  @prefix ex: <http://example.org/> .
  
  { ?person ex:name ?name } => { ?person ex:hasName true } .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const quads = new N3Reader().visit(cst);
```

## Backwards Implications

N3 supports backwards implications using `<=`:

```typescript
const input = `
  @prefix ex: <http://example.org/> .
  
  { ex:Bob ex:knows ex:Alice } <= { ex:Alice ex:knows ex:Bob } .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const quads = new N3Reader().visit(cst);
```

## Path Expressions

N3 supports path expressions using `!` (forward) and `^` (backward):

```typescript
const input = `
  @prefix ex: <http://example.org/> .
  
  ex:Alice ex:knows!ex:name "Bob's name" .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const quads = new N3Reader().visit(cst);
```

## Inverted Properties (is...of)

```typescript
const input = `
  @prefix ex: <http://example.org/> .
  
  ex:Bob is ex:knows of ex:Alice .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);

// Equivalent to: ex:Alice ex:knows ex:Bob .
```

## Nested Formulas

```typescript
const input = `
  @prefix ex: <http://example.org/> .
  
  {
    { ex:Alice ex:knows ex:Bob } => { ex:Bob ex:knows ex:Alice }

```

## QuadContext: Accessing Source Positions

For IDE features that need to associate positions with triples, use `readQuadContexts()` to get `QuadContext` objects. Each `QuadContext` contains the subject, predicate, and object with their source tokens:

```typescript
import { N3Lexer, N3Parser, N3Reader } from '@faubulous/mentor-rdf-parsers';
import type { QuadContext } from '@faubulous/mentor-rdf-parsers';

const input = `
  @prefix ex: <http://example.org/> .
  ex:Alice ex:knows ex:Bob .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const reader = new N3Reader();
const quadContexts: QuadContext[] = reader.readQuadContexts(cst);

for (const info of quadContexts) {
  console.log(`Subject: ${info.subject.value}`);
  console.log(`  Token position: line ${info.subjectToken.startLine}, column ${info.subjectToken.startColumn}`);
    
  console.log(`Predicate: ${info.predicate.value}`);
  console.log(`  Token position: line ${info.predicateToken.startLine}, column ${info.predicateToken.startColumn}`);
    
  console.log(`Object: ${info.object.value}`);
  console.log(`  Token position: line ${info.objectToken.startLine}, column ${info.objectToken.startColumn}`);
}
```

### Token Information

Each `QuadContext` provides RDF terms directly plus token metadata on `subjectToken`, `predicateToken`, and `objectToken`.

```typescript
const info = quadContexts[0];

// Get the exact text span for highlighting
const subjectSpan = {
  start: info.subjectToken.startOffset,
  end: info.subjectToken.endOffset,
  text: info.subjectToken.image
};

console.log(`Subject "${info.subject.value}" spans characters ${subjectSpan.start}-${subjectSpan.end}`);
```

### Working with N3-Specific Features

N3 supports formulas and variables. The `QuadContext` objects capture these correctly:

```typescript
const input = `
  @prefix ex: <http://example.org/> .
  
  { ?person ex:name ?name } => { ?person ex:hasName true } .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const reader = new N3Reader();
const quadContexts: QuadContext[] = reader.readQuadContexts(cst);

for (const info of quadContexts) {
    // Check for variables
  if (info.subject.termType === 'Variable') {
    console.log(`Variable: ?${info.subject.value}`);
    console.log(`  Defined at line ${info.subjectToken.startLine}`);
    }
}
```

# N3 (Notation3) Parsing

Parse [W3C N3](https://w3c.github.io/N3/spec/) documents with full CST access and RDF/JS quad generation.

## Basic Usage

```typescript
import { N3Lexer, N3Parser, N3Reader } from '@faubulous/mentor-rdf-parsers';

const input = `
  @prefix ex: <http://example.org/> .
  ex:Alice ex:knows ex:Bob .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const quads = new N3Reader().visit(cst);
```

## Formulas (Graph Literals)

N3 formulas are graph literals enclosed in `{ }`:

```typescript
import { N3Lexer, N3Parser, N3Reader } from '@faubulous/mentor-rdf-parsers';

const input = `
  @prefix ex: <http://example.org/> .
  { ex:Alice ex:knows ex:Bob } => { ex:Bob ex:knows ex:Alice } .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const quads = new N3Reader().visit(cst);
```

## Implications (Rules)

N3 supports logical implications using `=>`:

```typescript
const input = `
  @prefix ex: <http://example.org/> .
  @prefix log: <http://www.w3.org/2000/10/swap/log#> .
  
  { ?x ex:parent ?y . ?y ex:parent ?z } => { ?x ex:grandparent ?z } .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const quads = new N3Reader().visit(cst);
```

## Quick Variables

N3 quick variables use the `?` prefix:

```typescript
const input = `
  @prefix ex: <http://example.org/> .
  
  { ?person ex:name ?name } => { ?person ex:hasName true } .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const quads = new N3Reader().visit(cst);
```

## Backwards Implications

N3 supports backwards implications using `<=`:

```typescript
const input = `
  @prefix ex: <http://example.org/> .
  
  { ex:Bob ex:knows ex:Alice } <= { ex:Alice ex:knows ex:Bob } .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const quads = new N3Reader().visit(cst);
```

## Path Expressions

N3 supports path expressions using `!` (forward) and `^` (backward):

```typescript
const input = `
  @prefix ex: <http://example.org/> .
  
  ex:Alice ex:knows!ex:name "Bob's name" .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const quads = new N3Reader().visit(cst);
```

## Inverted Properties (is...of)

```typescript
const input = `
  @prefix ex: <http://example.org/> .

```

## QuadTokens: Accessing Source Positions

For IDE features that need to associate positions with triples, use `readQuadTokens()` to get `QuadTokens` objects. Each `QuadTokens` contains the subject, predicate, and object with their source tokens:

```typescript
import { N3Lexer, N3Parser, N3Reader } from '@faubulous/mentor-rdf-parsers';
import type { QuadTokens } from '@faubulous/mentor-rdf-parsers';

const input = `
  @prefix ex: <http://example.org/> .
  ex:Alice ex:knows ex:Bob .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const reader = new N3Reader();
const quadTokens: QuadTokens[] = reader.readQuadTokens(cst);

for (const info of quadTokens) {
    console.log(`Subject: ${info.subject.term.value}`);
    console.log(`  Token position: line ${info.subject.token.startLine}, column ${info.subject.token.startColumn}`);
    
    console.log(`Predicate: ${info.predicate.term.value}`);
    console.log(`  Token position: line ${info.predicate.token.startLine}, column ${info.predicate.token.startColumn}`);
    
    console.log(`Object: ${info.object.term.value}`);
    console.log(`  Token position: line ${info.object.token.startLine}, column ${info.object.token.startColumn}`);
}
```

### Token Information

Each `TermToken` in a `QuadTokens` provides:
- `term`: The RDF/JS term (NamedNode, BlankNode, Literal, Variable, etc.)
- `token`: The Chevrotain token with position information:
  - `startOffset`, `endOffset`: Character offsets in the input
  - `startLine`, `endLine`: Line numbers (1-based)
  - `startColumn`, `endColumn`: Column numbers (1-based)
  - `image`: The original text of the token

```typescript
const info = quadTokens[0];

// Get the exact text span for highlighting
const subjectSpan = {
    start: info.subject.token.startOffset,
    end: info.subject.token.endOffset,
    text: info.subject.token.image
};

console.log(`Subject "${info.subject.term.value}" spans characters ${subjectSpan.start}-${subjectSpan.end}`);
```

### Working with N3-Specific Features

N3 supports formulas and variables. The `QuadTokens` objects capture these correctly:

```typescript
const input = `
  @prefix ex: <http://example.org/> .
  
  { ?person ex:name ?name } => { ?person ex:hasName true } .
`;

const lexResult = new N3Lexer().tokenize(input);
const cst = new N3Parser().parse(lexResult.tokens);
const reader = new N3Reader();
const quadTokens: QuadTokens[] = reader.readQuadTokens(cst);

for (const info of quadTokens) {
    // Check for variables
    if (info.subject.term.termType === 'Variable') {
        console.log(`Variable: ?${info.subject.term.value}`);
        console.log(`  Defined at line ${info.subject.token.startLine}`);
    }
}
```
